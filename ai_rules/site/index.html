Why it matters

The real problem is not prompting. It is operating AI work at repeatable scale.

Most setups fail in the same places: the wrong tool gets used, rules live only in memory, delegated work vanishes into side channels, and nobody can reconstruct what happened later. This project exists to close those gaps with local routing, managed instruction files, and a session log pipeline that survives normal work.

Stop picking the model by feel

The routing layer makes provider choice explicit. Codex owns implementation, Gemini owns research and Google-stack work, and Claude owns review and presentation output.

Stop trusting memory for rules

Shared rule blocks are injected into instruction files per governed root, so naming, closeout, content structure, and CLI ownership are enforced where the work happens.

Stop losing the audit trail

The session logger writes a durable record to Notion, Google Docs, local archive storage, and SQLite, which makes retries and historical review possible.

System map

Four parts working together

The project combines a broker, a logger, managed instruction sync, and project-aware preflight. Each part is small on its own. The value comes from keeping them connected.

Session logging

Normalize payloads, submit to Notion and Google Docs, archive locally, and track state in SQLite.

Brokered delegation

Queue specialist work for Gemini or Claude when the task belongs on a different provider lane than the current terminal.

Managed rule sync

Load rule blocks from shared sources and upsert them into instruction files across governed roots.

Project-aware preflight

Resolve the active root, business context, and managed file coverage before work starts.

Routing

Three provider lanes, one ownership model

Global defaults live in global_cli_policy.yaml. If a project ships an AI_ROUTING.yaml, that local file becomes the source of truth.

Codex

Implementation lane

Default implementation and integration hub. Final repo-changing integration stays here unless intentionally overridden.

Gemini

Research lane

Research, Google-stack work, analytics, and content-heavy specialist roles.

Claude

Review lane

Review, refactor, hardening, and presentation-heavy output that benefits from a critique-first workflow.
