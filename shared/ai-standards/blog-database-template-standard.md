# Blog Database Template Standard

## Scope
Use this standard for any Notion database or calendar in this workspace that stages blog drafts for publication.

## Rule
Every brand blog staging database must have a native default Notion database template for new `Blog Post` entries.

## Required structure
- Keep scheduling and workflow state in database properties.
- Keep the actual draft in the page body of each database entry.
- New `Blog Post` rows should open with the page body already populated from the default template.

## Minimum template sections
- `Working Title`
- `Metadata`
- `SEO Title`
- `Meta Description`
- `Hook`
- `Outline`
- `Draft`
- `CTA`
- `Internal Links`
- `Assets Needed`
- `Editorial Notes`

## Multi-lane databases
- If one database stages posts for more than one brand or lane, add an explicit ownership property such as `Content Source` or `Brand Lane`.
- Do not rely on title naming alone to distinguish brand-owned versus personal content.

## Operational standard
- `Type` should identify `Blog Post` entries distinctly from social or newsletter entries.
- The default template should be applied only to blog-entry workflows unless another content type has its own template.
- Existing blank blog-entry pages should be backfilled with the same structure when possible.
